Add3DArchive()

Syntax

Add3DArchive(Pfad$, Typ)
Beschreibung
Fügt der aktuellen 3D Pfad-Liste einen neuen absoluten oder relativen Datei-'Pfad$' hinzu.

Alle 3D-Befehle, welche auf das Laden von Daten angewiesen sind (z.B. Texturen, Mesh's, Sky, World, ...), werden diesen Pfad benutzen. Es muss nach OpenScreen() oder OpenWindowedScreen() angegeben werden.

Sie wundern sich, warum es einen speziellen Weg zur Handhabung der Dateien für die 3D-Befehle gibt? Dies liegt daran, dass dieser den Datei-Zugriff flexibler macht und Sie auf die Daten mittels des Standard-Dateisystems oder direkt über die Archiv-Datei (wie .zip) zugreifen können. Dies ist ein sehr cooles Feature, da Sie all Ihre Daten in eine große Datei packen können, und dann auf deren Inhalte so zugreifen können als würden diese sich in einem Standard-Verzeichnis befinden.

Parameter

Pfad$ Der Dateipfad, welcher zur 3D-Pfad-Liste hinzugefügt werden soll. Dies kann ein tatsächlicher Pfad im Dateisystem oder ein ZIP-Archiv sein.
Typ Dies kann einer der folgenden Werte sein: 
  #PB_3DArchive_FileSystem  : Standard-Verzeichnis
  #PB_3DArchive_Zip         : Gepackte Zip-Datei

Rückgabewert

Diese Funktion gibt keinen Wert zurück.

Beispiel

  Add3DArchive("MyData.zip", #PB_3DArchive_Zip) ; Fügt die .zip Datei als ein Verzeichnis der Pfad-Liste hinzu
  LoadTexture(0, "MyTexture.jpg")               ; Lädt MyTexture.jpg aus der Zip-Datei
  LoadTexture(1, "World/Grass.jpg")             ; Lädt Grass.jpg aus dem "World\"-Verzeichnis der Zip-Datei
